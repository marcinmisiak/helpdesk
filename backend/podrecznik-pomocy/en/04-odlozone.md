# Deferred Tickets

The **Deferred** feature lets you temporarily set aside tickets you cannot deal with right now — and come back to them at the right moment.

## What Is Deferring a Ticket?

Deferring suspends work on a ticket until a specified date. The request remains active but disappears from the main list so it does not disrupt your workflow. On the chosen date it automatically returns to the view.

## How to Defer a Ticket

1. Open the ticket details
2. Click the **Defer** button
3. Select the return date
4. Optionally add a reason for deferring
5. Confirm

## Browsing Deferred Tickets

In the sidebar menu you will find the **Deferred** section showing the number of currently deferred requests. Clicking it takes you to a list of all deferred tickets with their return dates.

## Ticket Return

A ticket automatically returns to the active queue on the date specified when it was deferred. You can also manually restore a ticket earlier — click **Restore** in the deferred list.

## When to Use Deferring

- You are waiting for a response from the client or an external vendor
- The request requires action on a specific date
- You want to revisit the request after receiving additional information
- The ticket is blocked by another issue

> **Note:** Deferred tickets still count towards SLA metrics unless the administrator has configured otherwise.
